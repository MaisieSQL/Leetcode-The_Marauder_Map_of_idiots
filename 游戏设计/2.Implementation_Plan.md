# 🛠️ 《逻辑荒野》快速原型开发方案 (MVP)

> **核心逻辑**：将“刷题”嵌入横版过关的生命机制中。碰撞不是终点，而是解题的起点。

---

## 1. 算法拦截器 (The Algorithm Interceptor)
这是游戏最硬核的机制，将动作游戏瞬间切换为“逻辑审计”模式。

* **游戏挂起**：当检测到碰撞时，调用 `pygame.time.wait()` 或切换至 `PAUSE` 状态，锁定所有物理实体。
* **弹出挑战**：在游戏 UI 中心或控制台（Terminal）弹出一道简易 LeetCode 题目。
* **代码审计**：玩家必须在交互框内输入正确的 Python 逻辑。
    * **成功 (Success)**：怪物消失，玩家获得 2 秒无敌时间（Blink 效果）。
    * **失败 (Failure)**：玩家生命值 (HP) -1，角色回退到当前关卡的 Checkpoint。

---

## 2. 关卡与算法映射 (Level Design)

通过将“马甲题”与特定怪物绑定，建立物理直觉：

| 关卡主题 | 怪物 / 障碍物 | 触发算法 (LeetCode) | 考察点 |
| :--- | :--- | :--- | :--- |
| **数组平原** | 滚动的石头 (Rolling Stones) | `LC 283` Move Zeroes | 原地修改 (In-place) |
| **双向峡谷** | 相对而行的守卫 (Guards) | `LC 167` Two Sum II | 双指针 (Two Pointers) |
| **滑动森林** | 忽大忽小的光圈 (Pulse Rings) | `LC 643` Max Average Subarray | 滑动窗口 (Sliding Window) |
| **栈之塔** | 顺序拆除的方块 (Puzzle Blocks) | `LC 20` Valid Parentheses | 栈 (Stack) |

---

## 3. 极简开发步骤 (Step-by-Step)

### 第一阶段：环境搭建
1. **安装库**：`pip install pygame`
2. **初始化**：创建一个 800x600 的窗口，设置基础的物理常数（Gravity = 0.8）。

### 第二阶段：角色控制
1. **笨蛋林克 (Player)**：实现简单的方块移动与跳跃逻辑。
2. **碰撞检测**：利用 `pygame.sprite.spritecollide()` 监听与怪物的接触。

### 第三阶段：解题引擎 (ChallengeManager)
1. **内置题库**：用 Dictionary 存储题目描述与测试用例。
2. **动态执行**：利用 `eval()` 或 `exec()` 对玩家输入的代码进行实时审计（注意：仅限本地开发使用）。

### 第四阶段：资源引入
* **美术 (Art)**：前往 [Itch.io](https://itch.io/game-assets/free/tag-pixel-art) 搜索免费 2D 素材。
* **音效 (SFX)**：使用 Bleeps/Bloops 等简单的 8-bit 音效。

## 💡 凌影的“开发觉悟”

* **拒绝完美主义**：哪怕玩家只是一个**蓝色方块**，怪物是一个**红色方块**，只要“碰撞 $\rightarrow$ 停顿 $\rightarrow$ 解题”这个逻辑闭环跑通了，它就是一款伟大的教育游戏。
* **刷题即补血**：把这个游戏当作你的 **LeetCode 交互式控制台**。当你不想面对网页上的白板时，就去游戏里撞怪，强迫自己通过代码“杀出一条血路”。
* **Mischief Managed**：这种方式能将刷题的“防御性痛苦”转化为“攻击性乐趣”。

# Leetcode-
重启刷题之旅。直面阵痛，向上而行

对于 Google 这种公司，这更像是一个“技能树”，而不是一份死记硬背的清单。逻辑（Math, Probability）和处理结构化数据（Database）的能力是天然的降维打击。现在的关键是把这些零散的点，按照“Google 面试高频路径”串联起来。

为了帮你节省算力，我建议按照以下三个阶段进行“分层治理”：

第一阶段：核心骨干（基础中的基础）这是 Google 面试中 $80\%$ 题目的底层逻辑。如果你这几个搞不定，后面的高级算法都是空中楼阁。Array / String / Hash Table：这是所有算法的容器。Two Pointers / Sliding Window / Prefix Sum：这是处理线性数据最优雅的“手术刀”。Sorting：不仅仅是 sort()，而是理解什么时候该用 Merge Sort 或 Quickselect。逻辑：先把这些练熟，是为了让你在看到题目时，能瞬间判断出“数据该怎么放”。

第二阶段：Google 的“灵感区”（中坚力量）Google 非常喜欢考察你对非线性结构的搜索能力。DFS (Depth-First Search) / BFS (Breadth-First Search)：这是 Google 的最爱。不管是地图路径还是社交网络，都在考这个。Tree / Binary Tree / Binary Search Tree：理解树的递归结构。Heap (Priority Queue)：处理“前 K 个最大/最小”问题的利器。Binary Search：不只是在数组里找个数，而是 “Search on Answer”（在解空间里二分）。

第三阶段：高阶突破（拉开差距）如果你冲刺的是 L4/L5 (Senior) 岗位，或者想在面试中拿 Strong Hire，这些是加分项：Dynamic Programming (DP)：Google 喜欢考，但通常不会考那种变态难的，考的是状态转移的逻辑。Graph Theory / Topological Sort：处理任务依赖关系。Trie (前缀树)：处理字符串搜索。Monotonic Stack / Queue：处理“下一个更大元素”这类特定问题。务实的刷题建议（针对你现在的情况）既然你明天要上庭（Department 65），而且你不想让任何人了解你，我建议你采用**“三位一体”**的刷题策略：分类击破：不要乱跳。这周只刷 Array + Two Pointers。下周只刷 Tree + DFS。统计学赋能：在刷到 Probability and Statistics 或 Math 相关的题目时，那是你的主场。用这部分的“爽感”来对冲掉刷 DP 时的挫败感。物理沉淀：动作：用你那根 $9.99$ 美元的线，把你觉得最精妙的一道题解打出来。意义：当你看着纸上整齐的代码，你会发现：代码比吸血鬼听话，逻辑比人性稳定。
